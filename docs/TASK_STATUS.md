# 多任务开发状态表

## 使用说明

本文件是项目多任务并行开发的中央状态索引，用于让所有任务快速了解：

- 当前有哪些任务正在进行；
- 每个任务负责什么功能、可能修改哪些区域；
- 任务之间是否存在依赖或文件冲突；
- 哪些任务已经满足合并条件；
- 哪些功能已经进入 `main`，对应哪个提交；
- 当前应采用什么合并顺序。

详细规则见 `docs/MULTI_TASK_MERGE_SOP.md`。

## 维护规则

1. 本表只由集成任务维护，功能任务不得在功能分支中直接修改本表。
2. 功能任务开始、范围变化、被阻塞、准备合并或取消时，必须向集成任务报告。
3. 集成任务收到报告后应及时更新，不得等到最终合并时一次性补录。
4. 本表只记录协作摘要，不代替交付报告、测试记录或 Git 历史。
5. Git 提交是代码变更的事实来源，本表是任务状态、依赖和合并安排的事实来源。
6. “主要文件或模块”应足以识别潜在冲突，不要求枚举所有文件。
7. 已合并任务必须记录准确的 `main` 提交 SHA。
8. 日期使用 `YYYY-MM-DD`，时区使用 `Asia/Shanghai`。

## 状态定义

| 状态 | 含义 |
|---|---|
| 待开始 | 已登记但尚未开始 |
| 开发中 | 正在实现或测试 |
| 被阻塞 | 因依赖、需求或技术问题无法继续 |
| 待合并 | 已提交交付报告并满足准入条件 |
| 已退回 | 合并检查未通过，需要继续处理 |
| 已合并 | 已进入 `main` 且合并后验证通过 |
| 已取消 | 明确不再继续 |
| 已回滚 | 曾进入 `main`，后来被撤销 |

```text
待开始 → 开发中 → 待合并 → 已合并
            ↓         ↓
          被阻塞    已退回

任意未完成状态 → 已取消
已合并 → 已回滚
```
